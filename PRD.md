# H5 朋友圈分析助手 PRD (Product Requirements Document)

## 1. 项目概述 (Overview)

### 1.1 项目背景
用户希望通过分析潜在伴侣（"Crush"）的微信朋友圈内容，深入了解对方的性格、爱好、生活状态及潜在需求，从而制定有效的追求策略，最终达成建立恋爱关系的目标。

### 1.2 核心目标
创建一个**单页应用 (Single Page Application)**。用户在同一页面完成上传、分析等待及结果查看。系统通过分析上传的 5 张以上朋友圈截图，生成“追求异性高手”视角的专业建议。

### 1.3 核心 System Prompt
> "你是一个追求异性的高手，对于青春男女生的心理和外在表现，有非常强的洞察，也有一套很厉害的追求异性的技巧！擅长于输出简短但有效的分析和建议。"

---

## 2. 核心功能 (Core Features)

### 2.1 单页交互体验
- **极简界面**：去除繁杂的导航，聚焦核心功能。
- **动态流式体验**：上传 -> 分析 -> 结果展示，所有动作不跳转页面。

### 2.2 图片上传
- **批量上传**：支持一次性选择 5-20 张图片。
- **拖拽/点击上传**：支持移动端触摸选择及 PC 端拖拽。
- **UI 反馈**：上传成功后的缩略图预览。

### 2.3 豆包大模型分析 (AI Analysis)
调用 `doubao-seed-1.6-thinking` 模型，执行以下任务。

#### 2.3.1 输入
- 用户上传的截图内容（经过 OCR/Vision 提取的文本和场景描述）。
- **System Prompt**: 必须严格植入上述定义的“追求高手”人设。

#### 2.3.2 输出要求
需输出简短、犀利且有效的建议，结构如下：
1.  **一针见血的洞察**：用一句话概括对方的核心特质（例如：“这是一个渴望被理解但又习惯伪装高冷的猫系女孩”）。
2.  **核心需求分析**：对方现阶段最缺什么（安全感？新鲜感？陪伴？）。
3.  **Action Plan (行动必杀技)**：3 个具体的行动建议（简短有力）。

---

## 3. 页面布局与流程 (Single Page Flow)

页面分为三个垂直区域（Section），利用平滑滚动或状态切换展示：

### Section 1: 英雄区 (Hero & Upload)
- **Header**: Slogan "朋友圈懂你，更懂 TA。"
- **Upload Zone**: 占据屏幕核心位置。
    - 视觉元素：一个大而有质感的上传区域（支持虚线框、毛玻璃效果）。
    - 交互：点击上传 -> 选择图片 -> 预览图排列显示 -> 点击“生成攻略”按钮。

### Section 2: 分析状态层 (Analysis Overlay)
- 当点击“生成攻略”后，界面进入 **Loading 状态**（不跳转）。
- 显示**动态分析日志**（模拟“高手”的思考过程）：
    - "正在读取截图信息..."
    - "洞察潜意识心理..."
    - "制定专属追求策略..."

### Section 3: 攻略卡片 (Result Card)
- 分析完成后，直接在当前页面下方弹出或展开一张精美的**攻略卡片**。
- **内容布局**：
    - **顶部**：匹配度指数。
    - **中部**：高手洞察（Markdown 渲染文字）。
    - **底部**：行动建议清单。
    - **操作**：这部分内容支持一键复制或保存为图片。

## 4. 技术方案 (Technical Stack)

-   **框架**: React + Vite (极速构建单页应用)
-   **样式**: TailwindCSS (根据 Vibe Coding 理念，无需用户确认版本，直接使用最新版以实现高效开发与高级审美)
-   **状态管理**: React Hooks (useState) 处理简单的单页状态。
-   **AI 接入**: 
    -   前端模拟/真实调用后端 API。
    -   Model: `doubao-seed-1.6-thinking`。
